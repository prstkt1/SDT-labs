# Notes Service API

## 1. Варіант індивідуального завдання

- **N**=21,
- **V2** = (21 % 2) + 1 = 2,
- **V3** = (21 % 3) + 1 = 1,
- **V5** = (21 % 5) + 1 = 2,

- **Тематика:** Notes Service.
- **Спосіб конфігурації** Конфігураційний файл за шляхом /etc/mywebapp/config.json
- **Порт застосунку** 8080.
- **База даних** PostgreSQL.

---

## 2. Про застосунок (Notes Service)

Застосунок являє собою REST API сервіс, написаний на Node.js та TypeScript із використанням бази даних PostgreSQL.
Він підтримує механізм **Content Negotiation**:

- Якщо клієнт (наприклад, браузер) надсилає заголовок `Accept: text/html`, сервер повертає звичайну HTML-сторінку з таблицею.
- Якщо клієнт надсилає заголовок `Accept: application/json`, сервер віддає чисті дані у форматі JSON.

---

## 3. Документація API

| Метод  | Ендпоінт        | Опис                                                                 | Очікуваний формат |
| ------ | --------------- | -------------------------------------------------------------------- | ----------------- |
| `GET`  | `/`             | Головна сторінка зі списком усіх доступних маршрутів.                | `text/html`       |
| `GET`  | `/health/alive` | Перевірка життєздатності процесу сервера.                            | Plain text (`OK`) |
| `GET`  | `/health/ready` | Перевірка наявності підключення до бази даних.                       | Plain text (`OK`) |
| `GET`  | `/notes`        | Отримання списку всіх нотаток (ID та назва).                         | `json` / `html`   |
| `GET`  | `/notes/:id`    | Перегляд повної інформації про конкретну нотатку.                    | `json` / `html`   |
| `POST` | `/notes`        | Створення нової нотатки. Тіло: `{"title": "...", "content": "..."}`. | `json`            |

## 4. Розгортання та автоматизація

### Вимоги до Віртуальної Машини:

- **ОС:** Ubuntu Server 22.04 LTS або 24.04 LTS.
- **Ресурси:** Від 1 CPU, від 1024 MB RAM, мінімум 15 GB дискового простору.
- **Мережа:** Підключення через NAT із налаштованим прокиданням портів (Port Forwarding): хост порт `8080` -> гостьовий порт `80`.

### Як запустити автоматизацію:

Для швидкого розгортання проєкту на чистій ВМ використовується Bash-скрипт.

1. Склонуйте репозиторій:

   ```bash
   git clone git@github.com:prstkt1/SDT-labs.git
   ```

2. Запустіть скрипт розгортання:
   ```bash
   sudo chmod +x deploy.sh
   sudo ./deploy.sh
   ```

### Що робить скрипт:

1. Оновлює пакети та встановлює Nginx, PostgreSQL, Git, Node.js (v20).
2. Створює базу даних `notes_db` та налаштовує паролі.
3. Створює системного ізольованого користувача `mywebapp`.
4. Завантажує код з GitHub у директорію `/opt/mywebapp`, встановлює залежності та компілює TypeScript.
5. Налаштовує конфігураційний файл у `/etc/mywebapp/config.json`.
6. Створює та запускає systemd-юніт `mywebapp.service` із автоматичною міграцією БД перед стартом.
7. Налаштовує Nginx як Reverse Proxy.

## 5. Безпека та Адміністрування

### Користувачі в системі:

- **`student`**: Основний користувач для розробки. Має повні адміністративні права (sudo).
- **`teacher`**: Користувач для перевірки роботи. Має повні адміністративні права. При першому вході вимагається зміна пароля.
- **`mywebapp`**: Системний користувач (`/bin/false`) без права входу в консоль. Від його імені працює Node.js застосунок (захист від виконання шкідливого коду).
- **`operator`**: Користувач для обслуговування сервісу. Налаштовано суворі обмеження у файлі `/etc/sudoers`. Має право виконувати `sudo` **тільки** для команд: `start`, `stop`, `restart`, `status` для сервісу `mywebapp`, та `reload` для `nginx`.

### Обмеження Nginx:

Сервер Nginx налаштовано як Reverse Proxy. Він приймає зовнішні підключення на стандартному порту `80` і безпечно проксіює їх на внутрішній порт `5200`, де працює Node.js. Прямий доступ до порту `5200` ззовні недоступний, що забезпечує додатковий рівень безпеки та ізоляції застосунку.

## 6. Тестування системи

1. **Перевірка сервісу:** У терміналі ВМ виконайте `systemctl status mywebapp`. Сервіс повинен мати статус `active (running)`.
2. **Доступність:** Відкрийте браузер на хост-машині за адресою `http://localhost:8080`. Повинна завантажитися HTML-сторінка зі списком ендпоінтів.
3. **Створення даних:** Використайте Postman, Thunder Client або `curl` для відправки `POST`-запиту на `http://localhost:8080/notes` із JSON-даними для перевірки запису в БД.
4. **Перевірка прав:** Увійдіть у систему під користувачем `operator` та виконайте `sudo systemctl restart mywebapp` (має пройти успішно), а потім спробуйте виконати `sudo apt update` (має бути заблоковано системою безпеки).
